# CODER ロールガイド（Brief Gate 評価基準）

> このファイルは Division API の **Brief Gate（Leader Review Brief）** が
> Coder ロールの成果物を評価する際に参考にする基準書です。
> プロジェクト固有のルールはユーザーが自由に追記してください。
> Brief Gate は `.division/CODER.md` をロード対象として優先的に読み込みます。

## Coder の責務

1. 上流エージェントが渡した **既存コードベース** を尊重し、ゼロから書き直さない。
2. 最終出力には **必ず 1 つ以上のコードブロック** を含める。
3. ファイル本体は ` ```<lang>:<filepath> ` 形式で **完全な内容**（省略・「...」禁止）。
4. ツールコール JSON（`{"tool":"...","args":{...}}`）だけを返さない。
5. 元の指示・file-searcher の調査結果・Wave 4 集中再調査の Markdown を踏まえて差分を当てる。

## Brief Gate での評価チェックリスト（OK の条件）

- [ ] 三連バッククォートで囲まれた **コードブロックが 1 つ以上** ある
- [ ] 主要ファイルが ` ```<lang>:<filepath> ` 形式で書かれている
- [ ] 既存ファイルパス・既存コンポーネント名・既存ディレクトリ構造が**維持**されている
- [ ] 「Let me look at...」「まず○○を調査します」などの**前置き調査宣言だけで終わっていない**
- [ ] ツールコール JSON（`{"tool":...}`）が応答の主要部分を占めていない
- [ ] file-searcher の指摘した変更対象に対して**最低 1 ファイルは差分が当たっている**
- [ ] import / export / 型定義など、最低限の構文が壊れていない
- [ ] 新規ファイルを作る場合は **同一ディレクトリに既存があるなら**そちらを優先して編集している

## Not OK と判定すべきパターン（Brief Gate で差し戻し）

- コードブロックが 0 個 → 自動エラー扱い（再実行）
- ツールコール JSON だけ（`{"tool":"list_directory",...}` 等）→ 再実行
- 「以下省略」「...」「変更なし部分は割愛」などの省略表記
- ファイル全体が**新規パス**で書かれており、上流の既存ファイルパスを無視している
- file-searcher の指摘領域への変更が**全く反映されていない**
- 言語が指示と違う（日本語要求なのに英語コメントだけ等）
- セキュリティ的に明白な後退（`apiKey` のハードコード追加、`eval` 導入など）

## Brief Gate が出すべき VERDICT/FEEDBACK/BRIEF の指針

- **VERDICT: OK** … 上記チェックリストをほぼ満たし、Reviewer に渡す価値がある
- **VERDICT: Not OK** … チェックリストの致命項目（コードブロック無し / ツールコール JSON のみ / 既存パス全無視 など）に当てはまる
- **FEEDBACK** … Coder に「次こう直して」と指示できる粒度で、欠けている観点を箇条書きにする
- **BRIEF** … OK のときに Reviewer が短時間で評価できるよう、ユーザー要求の要点・実装概要・変更ファイル・確認してほしい観点を 4 セクションで書く
